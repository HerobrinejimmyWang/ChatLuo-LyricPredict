# Testcase

本文档用于定义 `LyricPredict` 的测试分级、预期行为和人工评测口径。测试目标不是证明模型具备开放域歌词生成能力，而是验证它是否能在个人数据集范围内稳定完成“续写一句歌词”这一任务，并在低置信度场景下选择不输出。

## dataset

- 所有测试默认基于训练集 `selflyricdata`。
- `selflyricdata` 已写入 `gitignore`，请勿以任何形式上传或分享，以保护歌曲版权；本项目仅用于个人使用。
- 非训练集输入只用于验证拒答策略，不用于评估泛化生成质量。
- 当前阶段优先验证三类能力：续写正确、低置信度拒答、格式边界稳定。

## 判定规则

- `无输出` 是合法结果，且在低置信度场景下优先视为正确保护行为。
- 默认只接受“下一句到第一个 `,` `.` `，` `。` 为止”的输出。
- 若输入末尾已带对应标点，则输出不再重复补同类前导标点；若输入末尾无停顿标点，则允许输出以 `，` 或 `。` 起始。
- `Level A` 关注干净输入下的稳定命中；`Level B` 关注轻微扰动下的容错与拒答；`Level C` 关注真实混合文本中的歌词抽取与续写。
- `Level B` 中的 `目标续写` 表示理想情况下应命中的歌词；`推荐行为` 只用于说明该输入下系统更应“输出”还是“无输出”。

## 格式约束补充

本节不单独升为新的等级，只用于统一人工评测边界和后续补测方向。

| 规则项 | 当前要求 | 建议补测方向 |
|--------|----------|--------------|
| 中文逗号停止 | 输出应在遇到 `，` 时停止或恰好输出到该分句边界 | 增加“输入末尾无逗号”与“输入末尾已有逗号”成对样例 |
| 中文句号停止 | 输出应在遇到 `。` 时停止 | 增加句末补 `。` 的歌曲样例 |
| 英文逗号停止 | 输出应支持 `,` 作为停止边界 | 增加含英文标点的歌词样例 |
| 英文句号停止 | 输出应支持 `.` 作为停止边界 | 增加含英文句号的歌词样例 |
| 输出长度边界 | 不应跨越两句或输出到第二个停顿后 | 人工检查是否出现“多输出一句” |
| 输出完整性 | 不应只给出半句残片 | 人工检查是否出现截断在非停顿位置 |
| 前导标点策略 | 输入末尾已有停顿标点时，输出不重复补同类标点 | 使用 A-1 第 7 条、A-2 第 3 条和 B-2 第 4 条联合复查 |

## 失败类型

| 类型 | 含义 | 处理建议 |
|------|------|----------|
| Pass | 输出与预期一致 | 计为通过 |
| Format Warning | 内容正确，但前导标点或空格形式与预期不一致 | 单独记录，不直接视为内容失败 |
| Boundary Fail | 输出跨过一句边界，或超过首个停顿符后仍继续生成 | 视为格式边界失败 |
| Abstain Fail | 预期应无输出，但系统仍输出内容 | 高优先级失败，需优先压低误报 |
| Recall Fail | 预期应输出，但系统给出无输出 | 记录为召回不足 |
| Content Fail | 系统输出了错误歌词或错误歌曲的后续 | 视为内容失败 |

额外说明：

- 在本项目中，`Abstain Fail` 的严重性高于 `Recall Fail`。
- `Level A` 优先看内容准确率，`Level B` 同时看容错与拒答，`Level C` 重点看真实场景下的抽取和续写是否都成立。
